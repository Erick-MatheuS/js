O que são eventos?

  • Por meio de JavaScript Podemos mapear algumas ações dos Usuários, que chamamos de eventos;
  • Como: movimento do mouse, click, mouse entrando ou saindo de um elemento, carregando da página e etc;
  • E então criar comportamento interessante como: animações de menu abrindo e fechando;

Eventos são procedimentos executados em consequência a uma ação. Por exemplo, quando o usuário clica em um botão, é disparado um evento deste elemento chamado “click”. Quando se pressiona uma tecla sobre outro elemento, é disparado um evento chamado “keydown” e quando a tecla é solta, o evento “keyup” é disparado.

Várias ações, não só na interface gráfica, fazem com que eventos sejam disparados. Então, sabendo que algo é feito quando certas ações são executadas, pode ser interessante e necessário ter controle sobre esta situação. Nesse ponto entra o TRATAMENTO DE EVENTOS, que consiste em definir o que será feito quando um determinado evento for disparado.

Algumas linguagens que possuem ambientes de desenvolvimento RAD (Rapid Application Development) permitem acessar esses eventos de forma bastante prática, através da interface gráfica do IDE. Porém, às vezes é preciso controlar esses eventos apenas pelo código, sem auxílio de ambiente gráfico, o que é muito comum para a linguagem Javascript. Por isso é importante conhecer a sintaxe da linguagem para esse tipo de ação.

O tratamento de eventos geralmente é feito a partir de funções nas quais se implementa o código que deve ser executado quando o evento ocorrer. Essas funções são chamadas de “event handlers” (tratadores de evento). A seguir veremos como efetuar o tratamento de eventos de elementos como botões e inputs HTML a partir de um script Javascript, garantindo uma maior interação com o usuário.

Evento Onload

Em JavaScript, esse evento pode ser aplicado para iniciar uma função específica quando a página é totalmente exibida. Também pode ser usado para verificar o tipo e a versão do navegador do visitante. Podemos verificar quais cookies uma página usa usando o atributo onload .

Em HTML, o atributo onload é acionado quando um objeto é carregado. A finalidade desse atributo é executar um script quando o elemento associado é carregado.

Em HTML , o atributo onload é geralmente usado com o elemento para executar um script assim que o conteúdo (incluindo arquivos CSS, imagens, scripts, etc.) da página da Web estiver completamente carregado. Não é necessário utilizá-lo apenas com a tag , pois pode ser utilizado com outros elementos HTML.

A diferença entre document.onload e window.onload é: document.onload dispara antes do carregamento de imagens e outros conteúdos externos. Ele é acionado antes do window.onload . Enquanto o window.onload dispara quando a página inteira carrega, incluindo arquivos CSS , arquivos de script, imagens, etc.

Evento OnBlur e OnFocus

onBlur: o evento onBlur ocorre quando um objeto perde o foco. ex: você tem 2 campos de textos para digitar seus dados. O primeiro você deve digitar seu nome e no outro o seu email. Após digitar o seu nome você terá que digitar o email, certo? Se você tivesse colocado um evento onBlur no campo onde digitou o nome, ao clicar no campo de email uma ação seria disparada.

onFocus: este evento ocorre quando o objeto recebe o foco. ex: quando você clica em um campo para digitar algo.

Event onChange

O onchangeevento ocorre quando o valor de um elemento HTML é alterado .

Dica: Este evento é semelhante ao evento oninput . A diferença é que o evento oninput ocorre imediatamente após a alteração do valor de um elemento, enquanto onchange ocorre quando o elemento perde o foco, apósa alteração do conteúdo. A outra diferença é que o evento onchange também funciona em elementos SELECT.
